About The Situation Room
Sources, setup, and how to use the dashboard
What this is

The Situation Room is a unified situational awareness dashboard that fuses news, hazards, policy, finance, research, and geo‑tagged signals into a single operational view. It is built for fast scanning, strong signal density, and resilience when individual feeds fail.

Where the data comes from

NOAA/NWS alerts, USGS earthquakes, NASA EONET & FIRMS, CDC travel notices, EIA energy data, BLS & US Treasury releases, CISA KEV, OpenAQ air quality, OpenSky aviation, CoinPaprika crypto markets, Polymarket, arXiv research, NOAA IncidentNews spills, and curated RSS sources (BBC, Guardian, PBS, Google News, etc.).

How to use it

Start with Global/US/Local scopes. Use the map to explore incidents and the Signal Deck for momentum. Search across categories, add custom tickers, and open the Imagery panel to toggle satellite/SAR/thermal overlays. Refresh on demand or change the interval in Settings.

Navigation & controls

The top bar is your command line: export a snapshot, force refresh, switch scopes, and tune the time window. Settings houses layout, theme, API keys, and help.

Keys you may need

OpenAI (AI analysis + query translation) is BYO and entered in Settings. EIA, data.gov, OpenAQ, and NASA FIRMS are server‑managed via the proxy layer.

Resilience

Each feed fetches independently with timeouts, caching, and fallback proxies. If a source fails, the rest of the dashboard continues to update. Feed health is surfaced in Settings and the Signal Deck. Static mode uses the latest published snapshot on GitHub Pages.
